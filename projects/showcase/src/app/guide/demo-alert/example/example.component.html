The alert service can be used to open alert dialogs with design styling.

Messages and titles will be translate by the TranslateModule.

You could controll alert instance's lifetime, by passing ttl parameter to alert function.

By designed if your ttl is setting to -1, it'll nerver closed.

You could set alert's title, by passing title parameter to alert function.

Your title and alert's message will auto translate by ngx-translate.

If you don't want alert could be close manually, set the dissmisable parameter to false.

You could add events for user to interact, by given parameter otherActions to alert functions.

Notifications will fire if the alert is closed.

Specifying global configuration defaults

Default alert options can be specified by providing an instance of AlertOption for ALERT_DEFAULT_OPTIONS in your application's root module.
